    CNNs & ResNet¶

    CNN pipeline¶

    • Input: raw image → Conv+ReLU (slide filter, Σ w·x + b → feature map) → Pooling (max pool, halves spatial size, downsampling) → Conv+ReLU (high-level patterns) → flatten → z = Wz → class scores → softmax
    • Output shape H_out = ⌊(H + 2P − F)/S⌋ + 1 — F=kernel size, P=padding, S=stride
    • Receptive field R_ℓ = R_{ℓ−1} + (k_ℓ − 1)·Π_{j<ℓ} S_j (init R₀=1)
    • Counting parameters: conv layer (k×k×C_in + 1)·C_out; fully connected input_size × output_size + bias
    • Inductive biases: translation equivariance, locality (nearby pixels correlated), parameter efficiency
    • ViT (Vision Transformer): split image into p×p patches → flatten → linear project → tokens. No inductive bias, global receptive field, needs more data

    ResNet¶

    • Why ResNet? Deep nets suffer ① vanishing gradients (early-layer derivatives tiny → slow update) and ② accuracy degradation (deeper → higher training error). Skip connections fix degradation
    • Residual: learn the difference between desired output and input, y = F(x) + x (F learned by weighted layers)
    • Gradient flows through identity: ∂L/∂x = ∂L/∂y·(∂F/∂x + 1) — the +1 keeps gradient alive
    • Architecture: x → Conv → BatchNorm → ReLU → Conv → BatchNorm, then y = ReLU(F(x)+x)
    • BatchNorm: per mini-batch, normalize with batch mean/variance (μ_B, σ_B²)

    Interview Q&A¶

    What problem do residual connections actually solve?

    The degradation problem: plain deep nets get higher training error than shallower ones (an optimization issue, not overfitting). The identity skip lets a block learn a residual F(x) and passes gradient through unchanged (the +1 term), making very deep nets trainable.

    CNN vs ViT inductive bias?

    CNNs bake in locality + translation equivariance + parameter sharing, so they're data-efficient. ViTs have almost no spatial inductive bias (global attention over patches) → more flexible but need much more data (or pretraining) to match CNNs.

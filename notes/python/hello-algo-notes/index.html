    Ch1 Encounter Algorithms¶

    1. Input -> Algorithms -> Output

    2. Data Structure: A process full of trade-offs

    3. Data structures are the foundation of algorithms. Data structures provide algorithms with structured storage of data and methods for operating on data.

    4. Data structures store data information, algorithms solve specific questions
    5. Algorithms can be implemented based on different data structures, but execution efficiency may vary greatly. choosing the appropriate data structure is key

    Ch2 Complexity Analysis¶

    1. Algorithm design:
      • Find a solution to the problem
      • Seek optimal solution: time efficiency and space efficiency
    2. repeatedly executing a task is very common.

    Iteration: solving bottom up¶

    1. for loop: suitable for use when the number of iterations is known in advance we usually use "for i in range(a, b) because it is a iteration variable time complexity is a linear relationship with n
      • for loop in python automatically handle i += 1
    2. while loop: you need to initialize condition variable i by yourself, the a in range, change i by yourself while has better flexible because we can adjust the condition variable by ourself like twice per round.
    3. nested loop: nest one loop inside another quadratic relationship, n^2

    Recursion: solving top-down¶

    • Having a function call itself. Consists of two phases
      1. Descend: The program continuously calls itself deeper, usually passing in smaller or more simplified parameters, until reaching a "termination condition"
      2. Ascend: When trigger the condition, the program returns layer by layer from the deepest recursive function, aggregating the result of each layer
    • Implementation perspective. Consist of three elements:
      1. Termination condition
      2. Recursive call: corresponds to descending, where function calls itself with smaller parameters.
      3. Return result, corresponds to ascending
    • Call stack: function's context data is stored in a memory area called "stack frame space"; consumes more memory space than iteration
      • function calls incur additional overhead: to call a function, the computer will do the following work hidden, thus is also less time-efficient than loops
        • Set up the call stack — save where you came from
        • Pass arguments — copy the values into the function
        • Allocate memory — create space for local variables
        • Return — clean up and go back to where you came from
      • Before termination triggered, there are n unreturned recursive exist simultaneously, with a recursion depth of n.
    • Tail Recursion: if a function makes the recursive call as the very last step before returning, the function can be optimized by the compiler to have space efficiency but not automatically support by compiler (eg: python), so still risk stack overflow issue
    • Recursion Tree: Used a lot when using "divide and conquer" subproblem decomposition
      • One call produces two call branches
    • "Last-in, first-out" of stack

    Time Complexity¶

    It counts the growth trend of the algorithms's runtime as the data volume increases - Asymptotic upper bound of function: how to find upper bound f(n)? 1. Count the number of operation - Ignore constants in T(n) - Omit all coefficient - Use multiplication for nested loops - Ranking - Constant O(1) - Linear O(n) - Quadratic O(n^2) - Exponential O(2^n) - Logarithmic O(logn): Reducing to half each round. To be precise, dividing into m corresponds to a O(log_m n) = O(log_k n/ log_k m) = O(log_k n) so m can be omitted

    Space Complexity¶

    • Input space
    • Temporary space
      • Temporary data
      • stack frame space
      • instruction space
    • Output space
    • Only focus on worst-case space complexity
      • Based on the worst input data
      • based on the peak memory during algorithm execution
    • Ranking:
      • O(1)
      • O(n): arrays, linked lists, stacks, queue, recursion
      • O(n^2): matrices and graphs
      • O(2^n): Binary trees
      • O(logn)

    Ch3 Data Structure¶

    • Arrays, linked lists, stacks, queues, hash tables, trees, heaps, and graphs.

      • Logical structure: data is arranged in a certain order
        • Linear: one-to-one sequential relationship
        • Non-linear
          • Tree structures: elements have a one-to-many relationship
      • Physical structure: reflects the way data is stored in computer memory Memory resources are important consideration, if there is a lack of contiguous large memory blocks, the data structure must be able to be stored in dispersed memory spaces
        • Contiguous space storage (arrays) Memory space for linked list nodes are one for value one for pointer All data structures are implemented based on arrays, linked lists, or a combination of both. Stack and queues can be implemented using either ways, hash tables may include both.
      • Linked list: can change the length, "dynamic data structure"
      • Array: cannot change length, "static data structure"

    • Basic data type:

      • CPU can directly operate on
        • Integer type: byte, short, int, long
        • Floating-point types: float, double, used to represent decimal numbers
        • Character: char
        • Boolean: bool
      • Binary form in computer: one binary bit is 1 bit. 1 byte consists of 8 bits. Each programming language has its own data type definitions

    Ch4 Array and Linked List¶

    Array¶

    • without initial values: start with 0s
    • with given initial values
    • Assessing elements is efficient, O(1)
    • Inserting elements/ Removing element: very inefficient, need to shift all elements related and will have loss of elements or memory waste
    • traverse arrays: can traverse by index or directly iterating through each element. enumerate() a function that automatically helps you loop through an iterate, keeping track of the index of the item for you
    • Advantages:
      • High space efficiency
      • O(1) for random access any element
      • Used for sorting and searching algorithms like quick sort, merge sor, binary search....; lookup tables

    Linked List¶

    Node + reference allows nodes to be stored scattered throughout the memory - Head node - Tail node: points to null, nullptr, 'None' in Java, C++, Python - O(1) for add and remove - O(n) for accessing a node - Common types: - Singly linked list - Circular linked list: Round-robin scheduling algorith - Doubly linked list: has references for both directions, requires more memory space - Used for stacks and queues; hash tables; graphs - You can use for fruit in fruits
